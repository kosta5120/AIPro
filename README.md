# Social-Media RAG Pipeline

A Retrieval-Augmented Generation pipeline over a corpus of ~17,000 Facebook,
Instagram, and Twitter posts (with a configurable sampling cap). Uses
**Chroma** as the vector store, **`sentence-transformers/all-MiniLM-L6-v2`**
for embeddings, and **`gemma3:4b`** via **Ollama** for answer generation.

## Project layout

```
data/
  raw/                       # input JSON dumps (Facebook / Instagram / Twitter)
  processed/                 # Chroma persistent store + chunks.jsonl
  MANIFEST.md
src/
  build_index.py             # build (or rebuild) the Chroma index
  utils.py                   # loading, cleaning, chunking
  retrieval.py               # Retriever class
  generation.py              # Ollama-backed generation + citation parsing
  rag_system.py              # exposes `answer(question) -> dict`
  rag_sysytem.py             # compatibility shim (same module)
eval/
  build_gold_set.py          # LLM-generate one Q+A per indexed chunk
  gold_set.jsonl
  run_eval.py                # Hit@k, MRR@k, ablation, sample answers
requirements.txt
README.md
```

## Required interface

```python
from src.rag_system import answer
answer("Who posted about NYFW?")
# -> {"answer": "...", "sources": ["fb_61450"], "retrieved_chunks": [...]}
```

`sources` is the list of chunk_ids actually cited by the LLM (parsed from
`[chunk_id]` markers in its reply). If no citations are detected we fall
back to all retrieved chunk_ids.

## Install

```powershell
pip install -r requirements.txt
```

You will also need **Ollama** running locally with `gemma3:4b` pulled:

```powershell
ollama pull gemma3:4b
ollama serve   # if not already running
```

The pipeline works without Ollama too: retrieval is fully functional, and
`answer()` will return retrieved chunks with a clear "generation unavailable"
message instead of an LLM answer.

## Build the index

```powershell
# Default: fixed-size word chunks, 500 words with 80 overlap, 1500 posts/source.
python src/build_index.py

# Sentence-based chunking.
python src/build_index.py --chunk-strategy sentence --chunk-size 400

# Quick smoke test (30 posts/source).
python src/build_index.py --max-per-source 30 --clean
```

CLI flags: `--chunk-strategy {fixed,sentence}`, `--chunk-size`, `--overlap`,
`--max-per-source`, `--collection-name`, `--persist-dir`, `--clean`.

The script is reproducible: deleting `data/processed/chroma` and rerunning
produces the same index.

## Ask a question

```powershell
python src/rag_system.py "Who posted about NYFW?"
```

## Evaluate

```powershell
# Regenerate gold_set.jsonl (one Q+A per chunk via Ollama, ~3970 LLM calls).
# Use --limit N for a smoke test, or --resume to continue an interrupted run.
python eval/build_gold_set.py --limit 20
python eval/build_gold_set.py            # full run (long; needs Ollama up)
python eval/run_eval.py                  # builds 2 ablation indices and prints metrics
```

`run_eval.py` prints a markdown table to stdout and writes it to
`eval/results.md`. It also samples 10 generated answers (skipped if Ollama
is down) and writes them to `eval/answers_sample.jsonl` for manual review.

## Chunking strategies

* **fixed** (default): word-level sliding window with overlap. Cheap,
  predictable, and works well for short social posts.
* **sentence**: greedy sentence packing up to a max-words budget with
  one-sentence overlap. Preserves natural boundaries; useful for longer
  Instagram captions.

## Gold set + evaluation

`eval/gold_set.jsonl` is generated by `eval/build_gold_set.py`: one
question + reference_answer per indexed chunk in
`data/processed/chunks.jsonl`, produced by the local LLM (`gemma3:4b`
via Ollama). Categories `factual`, `numerical`, `temporal`,
`comparison`, and `negation` are cycled deterministically across
chunks; the model may fall back to `factual` when a chunk does not
support the requested target category. `must_cite_chunk_ids` holds the
source `chunk_id` (e.g. `fb_61450_chunk_0`); the evaluator matches on
either `chunk_id` or its `doc_id` prefix, which keeps the metric
stable across chunking configurations.

The script writes one line per chunk and `flush()`es after each, so
runs interrupted with Ctrl+C can be resumed via `--resume`. The first
overwrite is preceded by a one-time backup to `eval/gold_set.jsonl.bak`.

Metrics reported: **Hit@k** and **MRR@k**, plus a chunk-size and top-k
ablation.

## Notes / limitations

* The corpus is social-media content (political / lifestyle / influencer
  posts) collected from public APIs. See `data/MANIFEST.md`.
* The LLM must be run locally via Ollama; no API keys required.
* Filenames `rag_system.py` and `rag_sysytem.py` both work (`rag_sysytem`
  is a shim that re-exports `rag_system`).
